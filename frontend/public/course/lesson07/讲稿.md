## 第一部分：Scaling Law——大模型最深刻的发现（10分钟）

### 故事的开始

2020年，OpenAI发了一篇论文，标题平淡得不像有什么了不起：**《Scaling Laws for Neural Language Models》**——神经语言模型的规模定律。

这篇论文做了一件简单的事：训练了几十个不同大小的Transformer，从几百万参数到几十亿参数，记录每个模型的最终损失（loss）。然后画图。

**他们发现了一个让所有人震惊的规律**。

【看第一节的图表】

横轴是训练总算力（FLOPs），纵轴是模型最终的loss。在**对数坐标**下，所有数据点几乎完美地落在一条直线上。

【点击"线性坐标"按钮】

切到线性坐标看不出规律。但**在对数坐标下，规律一目了然**——这是一条直线。

### 这条直线意味着什么

直线意味着：**loss和算力之间存在一个幂律关系**。具体公式是：

```
L = a · C^(-b)
```

其中L是loss，C是计算量，a和b是常数。

翻译成人话：**算力翻10倍，loss就稳定下降一个固定比例**。

这个关系在跨5个数量级的范围内都成立——从GPT-2的小模型到GPT-3再到GPT-4，**每一步都精确符合这条公式**。

### 为什么这是"投资圣经"

理解一下这个发现的恐怖之处：

如果你训练一个GPT-3级别的模型花了1亿美元，得到了loss = 2.0。Scaling Law告诉你，**只要再花10亿美元（10倍），你就能得到loss = 1.7**。再花100亿，loss = 1.5。

**只要你愿意花更多钱，模型就会变得更好**。这条曲线没有看到尽头。

这就是OpenAI、Anthropic、Google敢持续投入几百亿美元的科学依据。**他们不是在赌，他们是在按照公式投资**。

更恐怖的是——这个规律告诉你，**只要算力跟得上，竞争对手按同样路径也能做出同样的模型**。所以国内追赶速度这么快不是奇迹，是Scaling Law的必然。

### 这个规律会不会失效

这是个开放问题。目前有两种说法：

**乐观派**（OpenAI、Anthropic）：曲线还会继续。GPT-5、GPT-6会在这条曲线上继续往下走。

**怀疑派**：训练数据快用完了。互联网上高质量文本是有限的，不可能一直翻10倍。Scaling Law可能在2025-2026年遇到数据墙。

但无论如何，**过去5年这条曲线被反复验证**。这是大模型行业最坚实的科学基础。

## 第二部分：Chinchilla定律——参数和数据要均衡（5分钟）

### 一个被忽视的细节

Scaling Law有个细节：模型表现取决于"参数×数据×算力"。但这三个东西怎么分配？

GPT-3时代的做法是：**尽量堆参数**。1750亿参数，只训练了3000亿token——参数和数据的比例是1.7:1。每个参数大约只见过2个token。

【看第二节的对比】

打个比方——这就像一个**有惊人天赋但只读过几本书的人**。容量很大，但内容稀薄。模型有大量参数没有充分利用。

### Chinchilla的颠覆

2022年DeepMind发了一篇论文，做了一个关键实验：**在固定算力预算下，怎么分配参数和数据才最优？**

他们训练了几百个模型，最后得出结论：**最优比例是 1:20**——每个参数应该见过约20个token。

按这个比例，DeepMind训练了一个叫Chinchilla的模型——**700亿参数，1.4万亿token**。结果Chinchilla**全面优于GPT-3**，而算力消耗一样。

这彻底改变了行业。从2022年开始，**所有团队都按Chinchilla比例训练**。

### 2024年的新发现

【看下面的小字】

Chinchilla比例（1:20）也在被刷新。**Llama 3又把比例推到了1:200**——700亿参数喂15万亿token。每个参数见过200个token。

DeepSeek、Qwen等也在往这个方向走。**"小模型+海量数据"成了新主流**。

为什么往这个方向走？**因为推理成本**。模型小、推理便宜、部署友好，企业更喜欢用。

### 这件事对我们的启示

我们做CRM应用，经常要选模型。这里有个反直觉的事实：

**参数大不一定更好用**。一个充分训练的70B模型，往往比训练不充分的175B模型更好。

这就是为什么开源圈子里**Llama 3 70B、Qwen 72B**比"参数更大但训练数据少"的国产闭源模型更受欢迎——它们的"参数效率"更高。

**选模型时不要只看参数量，要看每个参数见过多少数据**。这是个老到的判断。

## 第三部分：涌现能力——非线性的能力跃迁（8分钟）

【滚动到第三节，深色背景的章节】

Scaling Law告诉我们loss会平滑下降。但**很多具体能力的提升不是平滑的——是突然的**。

这就是**涌现能力**（Emergence）——模型在小规模时完全不会某些任务，规模到达某个临界点后，**能力突然出现**。

### 看演示

【点击"三位数加法"】

横轴是模型参数规模，纵轴是该任务的准确率。

看这条曲线——参数小于10亿时，准确率几乎是0。模型**根本不会做三位数加法**。然后在某个临界点（约150亿参数）附近，**准确率突然跃升到80%以上**。

这种"S型曲线"的跃迁，就是涌现的典型形态。**没有任何过渡，能力一夜之间出现**。

【点击"语义理解"】

不是所有任务都涌现。语义理解从小模型时代就有部分能力，**规模化只是不断提升**。这是平滑曲线。

【点击"指令遵循"】

指令遵循（按用户复杂指令执行多步任务）在500亿参数附近出现明显跃升。**这是Agent能力的基础**——指令跟不准，Agent就跑不起来。

【点击"多步推理"】

多步推理是**最晚涌现的能力之一**——千亿参数才开始显著起色。这也是为什么GPT-4级别的模型才真正能做复杂的Agent任务。

### 涌现的工程意义

涌现现象给我们一个非常重要的工程指导：

**任务能力和模型规模可能是非线性关系**。

举例：
- 做需求分类——平滑型，小模型够用
- 做合同条款抽取——平滑型，中模型不错
- 做多步骤的需求分析+方案生成——涌现型，必须用大模型

**用错梯队的模型，效果会完全不一样**。

### 涌现的争议

【看那个黄色的"CONTROVERSY"块】

这里要给大家讲一个学术争议。**2023年Stanford发表了一篇论文**，标题挑衅意味十足：**《Are Emergent Abilities of Large Language Models a Mirage?》**——大模型的涌现能力是幻觉吗？

他们的论点：**很多"涌现"现象，是因为评估指标用了"全对才算对"的硬阈值**。

比如三位数加法，如果只有完全正确才算对，模型的准确率确实会从0突然跳到80%——因为答错一位就算0分。

但如果用"每位数字的正确率"这种**连续指标**，模型的能力提升其实是平滑的——它从"99%错"逐步变成"98%错"再到"50%错"再到"全对"。

这场争议没有完全结束。但有一个**实用共识**：

**无论涌现是真的还是评估幻觉，"模型大到一定程度突然变得好用"是真实存在的**。

这才是企业关心的事——**什么时候该升级到更大的模型**。

## 第四部分：2025年的模型梯队（5分钟）

【滚动到第四节模型梯队表】

理解了Scaling Law，我们来看今天市场上模型的梯队结构。这个表格对我们做技术选型直接相关。

### 三个梯队

**Frontier梯队**（最强）：
- GPT-4 / Claude Opus / Gemini Ultra
- 万亿参数级
- 适合：复杂推理、专业任务、Agent
- 成本：贵

**Strong梯队**（强力）：
- Claude Sonnet / GPT-4o / DeepSeek V3 / Qwen 72B
- 千亿参数级
- 适合：日常对话、代码、写作、RAG
- 成本：中等

**Efficient梯队**（高效）：
- Claude Haiku / GPT-4o-mini / Qwen 7B
- 百亿参数及以下
- 适合：高并发、批量任务、客服
- 成本：便宜

### 选模型的核心原则

**不是"越大越好"，而是"匹配场景才好"**。

举几个例子：

**场景1**：从客户访谈录音里抽取需求。这是一个相对结构化的任务，**Strong梯队就够了**。用Claude Sonnet或DeepSeek V3，成本可控、效果好。

**场景2**：基于历史项目数据，给客户的复杂需求设计完整方案。这需要**多步推理+知识整合**，是涌现型任务。**必须用Frontier**——Claude Opus或GPT-4。

**场景3**：批量处理几万条客户反馈，做情感分类。这是**简单任务大批量**，用**Efficient梯队**。一个Qwen 7B就够，成本能压到Frontier的1/100。

### 分层调度的实践

生产系统里**通常会混用2-3个不同梯队的模型**，按任务复杂度路由。

比如我们的CRM AI流程：
- **意图识别**：Efficient模型（Qwen 7B），毫秒级响应
- **需求抽取**：Strong模型（DeepSeek V3），准确率优先
- **方案设计**：Frontier模型（Claude Opus），质量优先

通过分层调度，**整体成本降低70%-80%，但用户体验仍是Frontier水平**。这是企业AI落地的核心工程问题——**不是用最强的模型，是用最合适的模型**。

### 中国模型的追赶

【看下面红色的"中国模型"专题】

曾经被认为差距三年的中国大模型，2024-2025年已经追到接近Frontier梯队。这背后是**Scaling Law的逻辑**——只要算力、数据、对齐都到位，结构上没有壁垒。

**DeepSeek**：671B总参数（MoE架构），推理能力对标Claude/GPT-4。最让行业震撼的是——**训练成本仅558万美元**。这挑战了"必须烧上亿才能做大模型"的常识。

**通义千问**：从7B到235B多个尺寸全开源。代码、数学、中文场景接近GPT-4水平。**企业私有化部署的首选之一**。

**豆包/Kimi/智谱**：各有侧重。差距已经从"代际"缩小到"细节"。

对我们做CRM实施的启示——**国产模型已经可以打一线项目**。客户对国产化、私有化的需求越来越强，这是巨大机会。

## 收尾（30秒）

总结今天三个核心：

1. **Scaling Law**：模型能力可以通过砸算力提升，关系是有数学公式的。这是过去5年大模型投资的科学依据。

2. **Chinchilla定律**：参数和数据要均衡。盲目堆参数不如喂更多数据。新的最佳比例可能是1:200。

3. **涌现能力**：能力跃迁是非线性的。某些任务在规模阈值前完全做不了，过了阈值突然学会。这意味着**模型梯队的选择极其关键**。

工程上的核心建议——**学会分层调度**。Frontier、Strong、Efficient三梯队混用，按任务复杂度路由。这是大模型时代企业落地的必修课。

## 备注：常见疑问

**Q: Scaling Law会一直有效吗？**
A: 这是开放问题。乐观派认为还会继续，悲观派担心数据墙。但即使遇到数据墙，合成数据、推理时计算（Test-time Compute）等新路径也在打开新的scaling方向。第19讲会展开。

**Q: 国产模型为什么能追这么快？**
A: 因为路径是公开的、清晰的——结构、训练方法、对齐技术都是开源论文。差距主要在算力获取（受出口限制）和数据规模上。但工程层面差距在持续缩小。

**Q: DeepSeek的"训练成本558万"是真的吗？**
A: 是真实的，但有上下文。558万是DeepSeek V3的"主训练阶段"成本，不包括前期实验、数据收集、对齐等。完整成本估计在5000万-1亿美元区间，但比GPT-4的几亿美元仍然低一个量级。这背后是MoE架构和工程优化的胜利。
