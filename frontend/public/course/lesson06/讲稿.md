**时长**：30分钟  
**目标**：让团队理解大模型从原始数据到能用产品的完整训练流程，理解为什么"开源Base模型"和"对话产品"是两个东西  
**配套演示**：demo.html（三阶段流程图 + 预测下一个词 + SFT前后对比 + RLHF偏好选择）

## 开场（2分钟）

上一讲我们打开了Transformer的引擎盖。今天换个视角——看看**怎么把一堆显卡和文本，炼成ChatGPT这样能用的产品**。

我先抛一个问题让大家想：

**为什么GPT-3在2020年就发布了，但直到2022年的ChatGPT才让世界惊艳？**

GPT-3和ChatGPT的底层模型结构几乎一样。差别在哪？答案就是今天要讲的——**对齐**。

GPT-3已经"会说话"了，但它**不愿意好好说话**。ChatGPT做的事情，是让模型"愿意好好和人说话"。这是一个完全不同的工程问题。

今天讲清楚现代大模型炼成的**三个阶段**：
1. **预训练**——学会接话
2. **监督微调（SFT）**——学会应答
3. **强化学习对齐（RLHF/DPO）**——学会好好说

打开 demo.html，跟我走。

## 第一部分：三阶段总览（4分钟）

【看顶部的三阶段流程图】

我做了一个简化的对比表格，先建立整体认知。三个阶段差异巨大：

**第一阶段：预训练**
- 数据：互联网级文本（万亿token）
- 算力：占总成本的99%
- 目标：学会"接话"——给一段文本，预测下一个词
- 产物：Base模型（基座模型）

**第二阶段：监督微调**
- 数据：几万到几十万条人工写的高质量问答
- 算力：占总成本的1%
- 目标：学会"应答"——把问答格式塞进模型
- 产物：Instruct模型（指令模型）

**第三阶段：强化学习对齐**
- 数据：几万到几十万条人类偏好排序
- 算力：不到1%
- 目标：学会"好好说"——回答符合人类偏好
- 产物：Chat模型（产品级模型）

【点击每个阶段查看详情】

注意算力分布——**99%的钱花在预训练，但1%的SFT和不到1%的对齐，决定了最终产品体验的80%**。这是个反直觉的事实，但理解了它就能理解很多大模型行业的怪现象。

## 第二部分：预训练——学会接话（10分钟）

【滚动到第二节预训练演示】

预训练的核心任务，简单到让人怀疑：**给一段文本，预测下一个词**。

就这么一件事。把维基百科、Common Crawl、GitHub、电子书全喂进去，每一段文本都用来训练"下一个词预测"。这叫**Next Token Prediction**——下一token预测。

### 看演示

【点击"中国的首都是"按钮】

模型给词表里每个词打了一个概率分数。"北京"的概率92%，其他词低得多。这是一个训练良好的模型应该有的行为——**它知道这个事实**。

【点击"机器学习的核心是"】

这个就是开放性问题。"从数据"31%，"让机器"24%，"通过"18%——分布更分散，因为好几种续写都合理。

【点击"今天天气真"】

这个完全是开放的——"好"42%，"不错"21%，"冷"13%，"热"10%。模型不知道你想说什么，就给出一个分布。

### 这件事为什么深刻

听起来"预测下一个词"特别简单对吧？但**这个简单任务逼迫模型学会了几乎所有人类知识**。

为什么？我们想想：

要预测"中国的首都是"后面的字，模型必须**记住"中国-北京"这个事实**——所以它学到了世界知识。

要预测代码的下一个字符，模型必须**懂编程语言的语法**——所以它学到了代码能力。

要预测一段对话的下一句，模型必须**理解上下文、人物关系、情绪**——所以它学到了对话能力。

要预测"质能方程是E="后面的字，模型必须**学过物理**——所以它学到了科学知识。

**"预测下一个词"是一个表面任务，背后逼着模型学会了一切**。这是大模型最深刻的秘密。

### 预训练的三个数字

【看下面的统计数字】

- **数据**：15万亿token级别（Llama 3用了15T，GPT-4据传更多）
- **参数**：700亿到几万亿
- **算力**：2.5万张H100显卡跑几个月
- **成本**：单次训练5000万到上亿美金

这些数字解释了为什么**预训练只有少数几家公司能做**——OpenAI、Anthropic、Google、Meta、字节、阿里、DeepSeek、月之暗面。其他公司几乎不可能从头预训练大模型，只能基于开源Base模型做微调。

### 预训练完成后的样子

预训练完成后的模型叫**Base模型**或**基座模型**。它**博学多才，但不会对话**。

你问Base模型"中国的首都在哪里？"，它可能给你接着写：

> "中国的首都在哪里？这个问题问得好。还有，日本的首都在哪里？韩国的首都在哪里？泰国的首都..."

为什么？因为它学的是"接话"——把你的问题当成一段文本的开头，然后接着写"看起来合理的下文"。它没学过"看到问题就回答"这个模式。

**所以Base模型不能直接当产品用**。它需要被驯化。这就是下一阶段——SFT。

## 第三部分：SFT——学会应答（7分钟）

【滚动到第三节SFT对比】

SFT全称Supervised Fine-Tuning——监督微调。它的目标很明确：**教Base模型用问答格式工作**。

### 看对比

左边是Base模型的反应，右边是SFT之后的反应。同样的问题"中国的首都在哪里？"：

**Base模型**：把问题当文本开头，接着写更多问题。
**SFT之后**：直接回答"中国的首都是北京"。

这个变化是怎么发生的？

### SFT的方法

方法非常直接：**用几万到几十万条人工写的高质量问答对，再训练一遍模型**。

每条数据形如：
```
{
  "user": "如何写好简历？",
  "assistant": "写好简历要注意三点：1. 突出量化成果...2. 匹配岗位关键词...3. 控制在一页..."
}
```

模型经过SFT后学会了几个关键能力：
- **识别"现在该回答了"**——看到USER后跟ASSISTANT
- **给出符合预期的格式**——结构化、清晰、可读
- **保持连贯的应答风格**——不再是"接龙"而是"对话"
- **覆盖各种类型任务**——写作、编程、分析、问答

### 关键：数据质量决定一切

**SFT数据的质量是开源模型差距的主要来源**。

OpenAI、Anthropic都雇了大量博士级别的人写问答对，覆盖编程、数学、写作、法律、医疗等各个领域。一条好的SFT数据可能要花一个博士半小时写。

国内做得好的开源模型（Qwen、DeepSeek）在SFT数据上的投入也极大。**Llama 3的Base模型和Qwen2的Base模型水平接近，但Instruct版本表现差异显著**——差距主要就来自SFT数据质量。

这给我们一个重要启示：**如果你想用开源模型做产品，选Instruct版本而不是Base版本**。除非你打算自己做完整的对齐流程——那是个非常昂贵的工程。

### SFT的局限

SFT之后的模型，已经能正常对话了，但还有几个问题：

**问题一：风格生硬**。回答可能像在背教科书——技术正确但读起来累。

**问题二：偶尔不安全**。可能回答危险问题、生成有害内容。

**问题三：缺乏共情**。在情感场景下表现冷淡。

**问题四：啰嗦或太短**。长度控制不好。

这些问题没法靠加更多SFT数据完全解决——因为它们涉及的是**"什么样的回答更好"这种主观偏好**，而不是"这个问题的答案是什么"这种客观事实。

需要新的方法。这就是最后一步——对齐。

## 第四部分：RLHF——学会好好说（7分钟）

【滚动到第四节RLHF演示】

RLHF全称 **Reinforcement Learning from Human Feedback**——基于人类反馈的强化学习。这是OpenAI发明的，**也是ChatGPT让世界惊艳的核心秘密**。

### 看演示

【看那个"和女朋友吵架"的例子】

同一个问题，模型生成了两个候选答案：

**Answer A**：列了4个解决步骤，引用了2019年的关系研究数据。专业、结构化、信息密度高。

**Answer B**：先表达共情（"先抱抱你，吵架确实让人难受"），再询问具体情况，最后给一个温和的建议。亲和、共情、像朋友。

**人类标注员会选哪个？大概率是B**。

但请注意——**A并没有"错"**。它的内容也合理、有依据。差别在于"风格更让人舒服"。

**这种差别没法用对错衡量，只能用偏好衡量**。这就是RLHF要解决的问题。

### RLHF的四步流程

【看下面的流程图】

**Step 1：收集偏好数据**。让模型对同一问题生成多个答案，雇人类标注员排序——A>B>C>D。

**Step 2：训练奖励模型**。用排序数据训练一个"打分模型"。给它一个问题和一个答案，它输出一个分数，预测人类会有多喜欢这个答案。

**Step 3：PPO优化原模型**。用强化学习算法（PPO是最常用的），把原模型调整到**最大化奖励模型的分数**。

**Step 4：反复迭代**。新的模型生成新的答案，再让人标注，再更新奖励模型，再训练。GPT-4据说迭代了上百轮。

### RLHF为什么有效

这里有一个深刻的洞察：**人类很难写出"最好的答案"，但人类很擅长比较两个答案哪个更好**。

写一个完美的答案需要专业知识。但比较两个答案哪个更让人舒服，**任何人都能凭直觉判断**。RLHF把这种"难以表达但容易判断"的偏好，转化成了模型可以学习的信号。

这是OpenAI的天才之处——**他们绕过了"什么是好答案"这个哲学问题，转而问"哪个更好"**。

### 2024年新趋势：DPO替代RLHF

RLHF流程复杂、训练不稳定、调参困难。Stanford在2023年提出了一个更简单的方法叫**DPO**（Direct Preference Optimization）。

**DPO直接用偏好数据微调，跳过了"训练奖励模型"这一步**。本质上把RLHF的多步流程压缩成了一步。

DPO的优势：
- 训练稳定，不容易崩
- 算力开销低
- 实现简单
- 效果接近甚至超过RLHF

**Llama 3、Qwen2、DeepSeek都在用DPO或它的变体**。可能未来对齐方法会完全转向这条路线。

但RLHF的核心思想——**用人类偏好数据训练模型**——是不会变的。变的只是具体实现方式。

## 第五部分：综合理解（3分钟）

【滚动到总结】

回顾整个流程：

> **预训练（学知识）+ SFT（学应答）+ 对齐（学偏好）= ChatGPT/Claude**

这三步缺一不可：
- 没有预训练，模型不会任何知识
- 没有SFT，模型不会问答格式
- 没有对齐，模型回答让人不舒服

### 几个对你工作有用的认知

**第一：选模型时的理解**

开源模型通常有两个版本：
- **Base**：只做了预训练。适合你自己做SFT和对齐
- **Instruct/Chat**：做完了所有阶段。可以直接用

我们做CRM应用，**99%的情况应该选Instruct版本**。除非你有团队和资源做完整的对齐流程——那是个非常昂贵的工程。

**第二：理解模型的"脾气"**

不同模型的"脾气"不一样——有的爱写长答案、有的爱用列表、有的更保守、有的更直接。**这些差异主要来自SFT和对齐阶段**，不是Base模型决定的。

所以同一个团队的不同模型版本（GPT-3.5 → GPT-4 → GPT-4 Turbo），脾气会很不一样。这是因为每个版本的对齐策略都在调整。

**第三：理解为什么benchmark分数和实际体验有差距**

很多人遇到过：**某个开源模型在benchmark上分数很高，实际用起来却不如GPT好**。

为什么？因为benchmark主要测知识和推理（预训练阶段决定），而日常使用体验主要由SFT和对齐质量决定。**真正的护城河往往不在Base模型，而在对齐**。

**第四：理解微调的边界**

经常有客户问"能不能基于我们的数据微调一个模型"。答案是要看微调什么：
- 微调"风格、格式、领域知识" → SFT，可行，成本低
- 微调"基础能力" → 几乎不可能，需要重新预训练
- 微调"价值观和安全" → 对齐，专业团队才能做好

大部分企业场景下，**RAG + Prompt Engineering比微调更实用**。微调只在有大量高质量领域数据、且需求非常垂直时才值得做。

## 收尾（30秒）

总结今天这一讲：

1. **预训练**：万亿token的"接话"训练，学会世界知识。占99%成本。
2. **SFT**：几十万条问答对训练，学会应答格式。
3. **对齐**：基于人类偏好的强化学习，学会"好好说"。这是ChatGPT的真正秘密。
4. **核心认知**：算力99%在预训练，但1%的SFT和对齐决定80%的产品体验。

## 备注：常见疑问

**Q: 我们公司能不能训练自己的GPT？**
A: 预训练需要几千张H100、几个月时间、上亿成本，几乎不可能。但基于开源Base模型做SFT和对齐，是可行的——成本可能在几万到几十万之间。这是大多数行业模型的真实做法。

**Q: 微调和RAG有什么区别？**
A: 微调改变模型本身的权重，RAG只在调用时给模型提供上下文。微调适合改变"风格、格式、能力倾向"，RAG适合处理"动态、海量、需要溯源"的知识。我们第10讲会详细讲。

**Q: 为什么RLHF这么复杂还要用？**
A: 因为它解决了一个根本性问题——把"难以表达但容易判断"的人类偏好转化成可训练的信号。但2024年DPO等更简单的方法正在替代它。
