## 第一部分：从文字到向量（6分钟）

### 神经网络只能吃数字

第一个问题：Transformer是个数学函数，它处理的输入是**向量**——一串数字。但你输入的是"今天天气真好"——这是文字。**怎么把文字变成数字？**

答案是两步：**分词（Tokenization） + 嵌入（Embedding）**。

### 演示分词

【在演示框输入"Transformer 是一个革命性的架构。" 点击"切分并向量化"】

看，这句话被切成了一组**token**。每个token都有一个唯一的ID。

注意几个细节：
- "Transformer"是一个英文词，作为一个完整token
- "是"、"一"、"个"是中文字符，每个一个token
- 标点也是token

**token是模型处理的最小单位**，不是字符也不是词。是个介于字符和词之间的概念。

### 为什么不直接按字符或按词切

【看下面的Insight块】

早期模型尝试过两种极端：

**按字符切**：词表小，但序列变得超长。"Transformer"按字母切是11个token，效率很低。

**按词切**：词表大，但遇到没见过的词就废了——"AlphaGoZero"、"GPT-4"、新造的网络词，模型完全不认识。

现代大模型用一种叫 **BPE（Byte Pair Encoding）** 的算法找平衡。简单说：从大量文本里统计什么字符组合最常见，把这些组合作为token。

- "the" 是个完整token
- "中国" 可能是一个token  
- "AlphaGoZero" 拆成 "Alpha" + "Go" + "Zero"
- 罕见的字符组合就退回到字符级别

GPT-3的词表大概5万个token，GPT-4扩到了10万左右，国产模型为了支持中文一般15-30万。

### 然后是嵌入

【看演示里的"Embeddings"那部分】

每个token还是个离散的ID（比如"是"是id 4567），网络怎么处理？

**第二步：嵌入（Embedding）**。

把每个token的ID映射成一个**高维向量**。GPT-3每个token映射到12288维向量，演示里我们简化展示16维。

【看那些彩色方块】

每个token变成一组数字。注意——**这些数字不是人写的**，是训练过程中自动学出来的。训练完成后，**语义相近的词在向量空间里靠得近**。

经典的例子：
- vector("国王") - vector("男人") + vector("女人") ≈ vector("女王")

向量之间的代数运算居然有语义意义——这是嵌入最神奇的地方。

我们做RAG用的Embedding模型（BGE、Qwen-Embedding），本质就是把这一步独立出来——给一段文字，输出它的语义向量。

### 小结

回到Transformer的视角：**输入处理就是先分词、再嵌入**。一句话经过这两步，变成一个矩阵——每行一个token的向量。

下一步是这一讲最微妙的环节：**怎么告诉模型词的顺序**。

## 第二部分：位置编码（5分钟）

### 一个被忽视的副作用

上一讲我们讲了Transformer的注意力机制——每个词同时看所有词。但这有个**致命副作用**：

> **它分不清谁前谁后**

注意力计算只看"词A和词B有多相关"，不管A在前还是B在前。所以对模型来说，**"我打你"和"你打我"是一样的**。

这显然不行。怎么办？

### 给每个位置一个独特的指纹

【滚到位置编码演示】

Transformer论文的方案叫**正弦位置编码（Sinusoidal Position Encoding）**。

简单说：**给每个位置生成一组独特的正弦/余弦值**，加到词向量上。这样每个词的向量里同时包含了"我是什么"和"我在第几位"两层信息。

【看那些彩色波形】

每条曲线是向量的一个维度。注意一个关键模式：

**低维度（dim 0、1）**：波长短、变化快——能精确区分相邻位置

**高维度（dim 60、61）**：波长很长、变化慢——刻画粗粒度的位置

这有点像时分秒——秒针变化快用来区分10秒和11秒，时针变化慢用来分辨上午和下午。**用不同频率组合刻画一个位置，就像用一组数字组成一个独特的指纹**。

### 拖动滑块体验一下

【拖"序列长度"滑块】

序列长度可以变化——位置编码的设计让模型可以处理**任意长度的序列**，包括训练时没见过的长度。

【拖"显示维度"滑块】

不同维度的波形频率差异巨大。**所有维度组合起来，每个位置都有一个独一无二的位置指纹**。

### 现代模型用什么

原始正弦编码已经被改进了。现在主流大模型用：

- **RoPE（旋转位置编码）**：Llama、Qwen、DeepSeek都用这个。把位置信息编码成旋转角度，更适合长序列
- **ALiBi**：直接在注意力分数上加位置惩罚

但**核心思想都没变**——给每个位置一个独特、可区分、能延伸到任意长度的"位置指纹"。

## 第三部分：QKV三件套（10分钟）

这是今天最重要的部分。注意力机制具体怎么算出来的？答案就是**Q、K、V三件套**。

### 一个直观的比喻

我用图书馆找书的比喻：

- 你想找一本"关于唐朝政治制度"的书——这是你的**Query（查询）**
- 图书馆每本书有书脊上的标签——这是**Key（钥匙）**
- 书的实际内容——这是**Value（值）**

你的Query在书脊标签（Key）上扫一遍，找最相关的几本。然后从这几本书里取出实际内容（Value）。

**Transformer里每个词同时扮演这三个角色**：作为查询者发出问题，作为被查者提供"标签"，作为内容提供者交出"实际信息"。

### 演示

【滚到QKV演示】

【点击"它"这个词】

看，当"它"这个词作为Query发起查询：

- 它的Q在问："我指的是谁？"
- 句子里所有词的K作为标签被检索
- 算出的注意力分数：**苹果55%**，其他都很低
- 最后输出的是这些Value的加权平均

**模型一次前向就完成了指代消解**——这在传统NLP里需要专门的模块。

【点击"红"这个词】

"红"是个形容词，它在描述什么？它的Q去找最相关的名词，注意力50%给"苹果"。

【点击"它"再看】

注意一个细节：**Q、K、V不是预设的角色，而是同一个词的向量经过三个不同的线性变换得到的**。也就是说，每个词同时持有自己的Q、K、V三套向量。

### 公式拆解

【看右下的公式】

```
output = softmax( Q · Kᵀ / √d ) · V
```

我们一步一步看：

**Q · Kᵀ**：矩阵乘法。每个词的Q去和每个词的K做点积，得到一个分数矩阵。点积反映"对齐程度"——越对齐分数越高。

**/√d**：除以维度的平方根。这是个数值稳定性技巧——维度太高的话点积值会很大，softmax后梯度容易消失。

**softmax**：把分数归一化成0-1之间的概率，所有分数加起来等于1。

**× V**：用归一化的分数作为权重，对所有词的V做加权求和。

**得到的output就是这个词在当前上下文里的新表示**——融合了所有相关词的信息。

### 为什么这么设计是天才的

让我多花一点时间讲这个，因为它太精妙了。

**第一**：Q和K的分离让模型可以学到非对称的关系。"它"对"苹果"的注意力很高，但"苹果"对"它"的注意力可能不高——这不矛盾，因为它们用的是不同的Q和K。

**第二**：QKV三套向量都是从原始embedding学习得到的**线性变换**。这意味着模型可以学到**针对不同任务的不同视角**——同一个词在Q视角下可能是"找代词指代什么"，在K视角下可能是"被指代的可能性"。

**第三**：**多头注意力（Multi-Head Attention）**。实际Transformer不是只算一次QKV，而是同时算很多组——每组叫一个"头"。GPT-3有96个头。每个头学到不同的关注模式：有的头专门看语法、有的头专门看语义、有的头专门看指代。**多头让模型同时从多个角度理解上下文**。

【看下方Insight块】

记住这句话：

> **Q和K是一对锁和钥匙，能配上的得到高分；V是钥匙打开后取走的内容**

## 第四部分：三种架构变种（5分钟）

### 原始Transformer 长什么样

Transformer论文里画的架构其实是一个**Encoder-Decoder双塔**——左半边Encoder，右半边Decoder。它最初是用来做机器翻译的。

但是后来人们发现，**两半可以拆开单独用**。这就有了三种架构变种。

### 演示三种架构

【点击"仅 Encoder"标签】

**仅 Encoder架构**：只用左半边。每个词可以**同时看到所有其他词**——前后都能看。

擅长：理解类任务——分类、抽取、相似度判断、检索

代表：**BERT**（Google 2018年）、RoBERTa、**所有Embedding模型**（BGE、Qwen-Embedding、OpenAI ada）

我们做RAG知识库的时候，把文档切片做向量化用的就是Encoder架构。它读完整段文字，输出一个"语义向量"。

【点击"仅 Decoder"】

**仅 Decoder架构**：只用右半边。每个词**只能看到自己和前面的词**——后面的词被"遮"住了，叫**因果掩码（Causal Mask）**。

擅长：生成类任务——从左到右一个词一个词写

代表：**GPT系列**、**Claude**、**Qwen-Chat**、**Llama**、**DeepSeek**——**所有聊天大模型本质都是这种**

为什么所有聊天模型都是Decoder？因为对话本质就是"生成下一个词"。给你一段开头，模型预测最可能的下一个词，然后把这个词加到输入里，再预测下一个，循环。

【点击"Encoder + Decoder"】

**Encoder + Decoder架构**：左右都用。Encoder完整理解输入，Decoder生成输出，中间通过**Cross-Attention**连接（Decoder的Q来自自己，K和V来自Encoder的输出）。

擅长：输入和输出形式差别大的任务——翻译（中→英）、摘要（长→短）、问答

代表：**T5**、**BART**

但是有意思的是——**这种架构在新模型里越来越少见了**。因为大家发现，纯Decoder通过更大规模也能做翻译和摘要。简单的架构 + 大规模数据 > 复杂的架构 + 中等数据。

### 一个简单的判断框架

记住这个对应：

> **Encoder = 理解（BERT、Embedding模型）**  
> **Decoder = 生成（GPT、Claude、所有聊天模型）**  
> **Encoder-Decoder = 转换（T5、BART，机器翻译）**

我们做CRM应用要做选型，比如：
- 做语义检索 → 用Encoder架构的Embedding模型
- 做文档抽取 → 用Decoder架构的大模型prompt
- 做对话机器人 → 用Decoder架构的聊天模型

理解了这层，选型就有了清晰的依据。

## 收尾（2分钟）

今天我们把Transformer拆开看了4个零件：

1. **分词与嵌入**：文字 → token → 向量
2. **位置编码**：给每个位置加上独特的"位置指纹"
3. **QKV三件套**：每个词扮演查询者、被查者、内容提供者三个角色，注意力本质是Q和K的点积
4. **架构变种**：Encoder理解、Decoder生成、双塔做转换

记住一个关键洞察——**今天所有大模型本质都是这4个零件的组合**。GPT-4比GPT-2大100倍，但底层就是更多层、更宽、更多注意力头的同一架构。**没有架构上的本质创新，只有规模和工程上的优化**。

这也引出了下一讲的核心问题：**既然架构都一样，为什么GPT-3只是个"接话机器"，而ChatGPT能聊天、写代码、回答问题？**

答案是**对齐（Alignment）**。下一讲我们讲清楚 **预训练 + SFT + RLHF** 这个三段式过程——这是把"能写"的模型变成"好用"的模型的关键一跃。

## 备注：常见疑问

**Q: 为什么GPT没有Encoder也能理解中文？**
A: Decoder在生成每个词时也会先"理解"前面的所有词——它的Self-Attention层和Encoder的功能很像，只是看不到后面的词。所以理解能力是有的，只是受限于"只能看左边"。

**Q: 多头注意力的"头"是什么？**
A: 把QKV的维度切成多份，每份独立算一次注意力。比如12288维切成96份，每份128维。各个头独立学习不同的关注模式。最后把96个头的结果拼起来。

**Q: 为什么Decoder架构能做几乎所有任务？**
A: 因为只要任务能用"输入文本→输出文本"表达，Decoder就能做。包括翻译、摘要、分类、问答——只要prompt设计好，输出格式定义好，都能用同一个Decoder完成。这叫**Prompt Engineering**，第9讲会讲。
