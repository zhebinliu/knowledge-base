01

从一段文字到一串数字

TOKENIZATION & EMBEDDING
神经网络只能处理数字，不能处理"今天天气真好"这种文字。所以第一步——把文字切成"token"，再把每个token变成一个向量。这一步叫分词与嵌入，是所有大模型的起点。

STEP 2 · EMBEDDINGS（每个token变成一个向量，这里展示前16维）
每个token会被映射到一个高维向量（GPT-3是12288维，这里只展示16维示意）。这些向量在训练前是随机初始化的，训练过程中会自动学习—— 让"国王"和"皇帝"这种语义相近的词，在向量空间里也靠得近。这个过程叫嵌入（embedding），是所有语义理解的基础。

为什么不直接按字符切？

早期模型按字符切，但中文一个字一个token、英文一个字母一个token，序列变得超长，效率低。

现代大模型用 BPE（Byte Pair Encoding）或类似算法，从大量文本中统计出最常见的字符组合作为token。 常见词整个作为一个token（"the"、"中国"），不常见的词拆成子词（"transformer" 可能拆成 "trans" + "former"）。 这是大模型词表通常在5万到30万之间的原因。

02

给每个位置一个独特的指纹

POSITION ENCODING
Transformer的注意力机制有个"副作用"——它把所有词同时看一眼，分不清谁前谁后。"我打你"和"你打我"对它来说一样。怎么办？给每个位置加一个独特的"位置指纹"。
DEMO · 位置编码的波形

每个位置 = 一组不同频率的正弦/余弦值

Transformer原始论文用正弦余弦函数生成位置编码。每个位置在每个维度上都有一个独特的值，低维度变化快、高维度变化慢——就像"时分秒"用不同频率表示一个时间。

每条曲线代表向量的一个维度。低维度（dim=0,1）波长短、变化快，能区分相邻位置；高维度（dim=60,61）波长长、变化慢，刻画粗粒度的位置。 把这组值加到token的embedding上，模型就既知道"这个词是什么"又知道"它在哪个位置"。

现在主流大模型用更先进的位置编码（RoPE、ALiBi），但核心思想都是给每个位置一个独特、可区分的"指纹"。
03

QKV 三件套：注意力的真正机制

QUERY · KEY · VALUE
上一讲我们看了"注意力分数"长什么样，但没讲它怎么算出来的。答案就是QKV——把每个词变成三个角色：查询者、被查者、被取的内容。这三个角色完美对应人脑的"检索"过程。

Q
QUERY · 查询
当前词作为发起方。它带着一个"问题"去问其他词："你和我有多相关？"
K
KEY · 钥匙
每个词作为被查询的"标签"。Q和K做点积，得到相关性分数——分数越高，越相关。
V
VALUE · 内容
每个词的实际信息。最终输出 = 所有V的加权平均，权重就是Q和K算出的分数。

为什么是Q×K而不是Q+K？

点积的几何意义是两个向量"对齐程度"——方向一致时点积大，方向不同时点积小，方向相反时点积为负。 这正好对应"相关性"这个语义。

一个词的Q和另一个词的K点积大，意味着"我在找的"正好是"你能提供的"。 所以Q和K相当于一对锁和钥匙——能配上的会得到高分，配不上的得低分。

实际计算中，所有词的Q、K、V都是同一句话过三个不同的线性层得到的。也就是说，每个词同时扮演查询者、被查者、内容提供者三个角色。

04

三种架构变种：怎么搭出BERT和GPT

ENCODER · DECODER · ENCODER-DECODER
原始Transformer是Encoder-Decoder架构（左半边Encoder、右半边Decoder）。但实际应用中我们经常只取一半——BERT是纯Encoder、GPT是纯Decoder。这两条路线决定了不同模型的"性格"。

一个简单的判断框架

Encoder擅长"理解"——它能同时看到所有词，所以适合分类、抽取、检索这类需要全局理解的任务。 我们做的语义检索、Embedding模型，都是Encoder架构（BGE、Qwen Embedding、BERT这些）。

Decoder擅长"生成"——它从左到右一个词一个词生成，每个词只能看到前面的词（叫"因果掩码"）。所有聊天大模型都是Decoder架构（GPT、Claude、Qwen-Chat这些）。

Encoder-Decoder适合"翻译"类任务——先用Encoder完整理解输入，再用Decoder生成输出。机器翻译、文档摘要、Seq2Seq任务用这种架构（T5、BART）。
