# 友发钢管业务现状调研纪要

> 文档类型: 业务现状调研访谈纪要 (Discovery Interview Notes)
> 调研周期: 2026-03-22 ~ 2026-03-28
> 调研方式: 现场访谈 + 流程观察 + 系统截屏
> 参与方: 实施咨询团队 3 人,客户 12 个部门 27 人
> 文档状态: 已经客户方业务总监确认

---

## 一、调研对象与覆盖

本轮调研覆盖 4 大业务线 12 个部门,访谈对象包括:

- 渠道管理部 (经销商管理科 / 价格政策科 / 区域销售支持科)
- 工程业务部 (大客户 / 投标管理 / 项目交付)
- 零售业务部 (终端门店 / 促销活动 / 物流支持)
- 营销支持部 (主数据 / 报表 / 客服 / 财务对接)
- IT 部 (ERP 运维 / 系统集成 / 数据)
- 财务部 (应收 / 应付 / 信用)

## 二、核心业务流程现状

### 2.1 经销商签约流程 (现状: 12-18 天,纸质 + 邮件 + 微信)

1. 区域销售经理上报新经销商信息 (微信群口头 + Excel 报表)
2. 渠道管理部初审 (3-5 天,人工核查工商信息、行业背景、信用)
3. 价格政策科核定授信额度 / 价格档级 (2-3 天,Excel 测算 + 邮件审批)
4. 渠道总监终审 (1-2 天,纸质审批)
5. 合同寄送 + 签字盖章 (4-6 天,顺丰邮寄)
6. 财务建档 + ERP 客户主数据创建 (1-2 天)

**痛点**:
- 纸质审批易丢失,平均每月 8-12 单经销商签约,丢件率 5%
- 价格档级核定靠 Excel + 个人经验,各大区标准不一,客户投诉"区域价格不公"
- 与 ERP 客户主数据脱节,经销商创建后 IT 部门要手动同步 1-2 天

### 2.2 大型工程项目跟进 (现状: 平均 14 个月,信息散落)

1. 线索来源: 行业展会 / 老客户介绍 / 政府招标网站 (无系统记录)
2. 项目立项: 大区销售经理评估后报项目经理 (Word 文档 + 邮件)
3. 技术方案: 解决方案部出具技术方案 (3-5 周,涉及材料 / 规格 / 物流方案)
4. 商务报价: 商务部出具报价 (1-2 周,Excel 报价模板 + 价格审批)
5. 投标: 技术标 + 商务标 (1-2 周,投标项目组现场组合)
6. 中标 → 合同 → 生产排程 → 发货 → 回款 (跨财务 / 生产 / 物流多部门)

**痛点**:
- 项目状态管理依赖项目经理个人 Excel 跟单表,管理层每月开会才能看到全口径项目情况
- 报价审批走邮件,常因审批人出差延误 3-5 天
- 中标后到回款过程中,商务 / 生产 / 物流 / 财务信息散在 4 个系统 + 6 个微信群
- 历史项目数据无沉淀,新项目报价时找历史参考靠个人记忆

### 2.3 终端门店动销采集 (现状: 几乎为零)

- 友发钢管的终端门店主要是建材市场的零售门店、五金店、批发市场摊位
- 当前模式: 经销商一票出货给门店,后续门店动销 / 库存 / 价格执行**完全不掌握**
- 总部知道经销商进了多少货,**不知道**门店卖了多少、卖给谁、卖什么价
- 唯一的数据来源: 大区销售经理出差时随机拜访门店的口头反馈

**痛点**:
- 经销商串货 (从低价区调货到高价区销售) 频发,总部无法预警
- 总部促销活动到不了终端,经销商截留促销资源
- 终端客户画像缺失,新产品上市无法精准触达

## 三、关键干系人观点

### 张志强 (营销总经理) — 项目发起人

- 最关心: 经销商串货预警、大型工程项目可视化看板
- 期望: "明年初业绩会上,我能在 PPT 里直接展示当前 200+ 个 100 万以上工程项目的实时进度"
- 担忧: 担心数字化推进太激进,一线销售抗拒,影响业绩

### 王立军 (CIO) — 项目负责人

- 最关心: 与 ERP 的集成稳定性、上线后维护成本
- 期望: "希望 CRM 不要变成第二个孤岛,要和 ERP / OA 拉通"
- 担忧: 经销商账号管理复杂,1200 多家经销商各自的子账号体系怎么映射进 CRM

### 李梅 (渠道管理总监) — 关键业务方

- 最关心: 经销商政策线上化、串货预警
- 期望: "审批不要让我等一周才看到一个签约"
- 担忧: 一线区域销售经理是否愿意把客户信息录入系统 (公司没有强制 KPI)

### 周建华 (工程业务总监) — 关键业务方

- 最关心: 项目甘特图、投标历史复用
- 期望: "新项目报价时,我希望系统能告诉我类似项目过去三年的报价区间"
- 担忧: 大客户的核心商务条款被系统记录后泄密风险

### 刘洋 (零售业务总监) — 关键业务方

- 最关心: 移动端易用性、经销商配合度
- 期望: "门店老板没耐心,小程序点 3 下就要能完成动销上报"
- 担忧: 5000 多家门店覆盖目标过于激进,首期 1500 家已是天花板

## 四、当前系统盘点

| 系统 | 用途 | 在用情况 |
|---|---|---|
| 用友 NC (ERP-U8 V12) | 生产 / 财务 / 库存 / 出厂 | 全公司核心系统,2018 上线 |
| 泛微 OA | 流程审批 / 公文 | 仅限办公审批,业务流程未上 |
| 自研订单系统 | 经销商订单录入 | 简陋,无审批 / 无报表 |
| Excel + 微信群 | 项目跟进 / 经销商管理 | 现状主力 |
| 钉钉 | 内部沟通 | 与微信群并行 |

**结论**: 友发钢管的业务系统在出厂以前覆盖度高 (ERP),出厂以后几乎空白 (CRM 是空地),这是本项目的机会窗口。

## 五、组织变革风险点

1. **区域销售经理可能抗拒**: 当前客户资源在个人手里,一旦录入系统等于"客户资源公司化",涉及个人利益
2. **经销商可能不配合动销上报**: 经销商不愿让总部掌握终端数据 (失去对总部的谈判筹码)
3. **IT 部门资源紧张**: 同时支持 ERP 升级和 CRM 实施,人力可能不足

## 六、调研结论与建议

1. 项目目标合理但激进,建议**门店动销首期目标从 1500 家调到 800 家**,降低首期上线压力
2. 强烈建议设立**项目变革管理小组**,由营销总经理直接领导,处理一线抗拒
3. 经销商体系复杂 (1200+ 家、4 类、跨 4 大区),建议**渠道模块单独立项**或拉长到 8 个月
4. ERP 集成接口建议**第三方中间件 (如 MuleSoft / EAI)**,避免点对点集成后期难维护
5. 终端门店小程序的 UI / 交互需做**专项可用性测试**,目标点击 3 下完成动销上报
