## 一、方案总览

本方案以"**先打地基、再起高楼**"为基本指导思想,围绕友发钢管"经销商 + 工程 + 终端"三大业务线和"客户主数据 + ERP 集成"两大基础设施,设计如下能力地图:

```
                    [营销决策看板 (高管层)]
                            │
   ┌─────────────┬──────────┴──────────┬─────────────┐
   │             │                     │             │
[经销商管理]  [工程项目管理]    [终端动销采集]    [客户与商机]
   │             │                     │             │
   │             ├── 项目机会库         ├── 小程序 (轻)
   │             ├── 投标管理           ├── 动销采集
   ├── 签约      ├── 报价审批           ├── 库存盘点
   ├── 政策      ├── 项目协作           └── 价格执行
   ├── 价格      └── 履约跟进
   └── 串货预警
                            │
                ┌───────────┴────────────┐
              [客户主数据]          [ERP 集成]
                            │
                       [基础平台: 组织 / 权限 / 工作流]
```

## 二、分模块方案要点

### 2.1 经销商管理模块

**核心能力**:
- 经销商全生命周期: 引入申请 → 资质审核 → 准入评估 → 签约 → 政策授权 → 价格档级 → 月度返利核算 → 退出
- 政策线上化: 价格政策 / 返利政策 / 区域保护政策线上配置 + 一键下发到经销商门户
- 串货预警引擎: 基于"经销商发货流向 + 终端门店动销地理位置"自动识别异常流转,T+1 输出串货预警清单
- 价格档级模型: 按"客户级别 (A/B/C) × 区域 (4 大区) × 产品大类 (3 类)"建立 36 个价格档级,自动套用

**关键决策**:
- 经销商门户复用"纷享销客经销商 BP 版"标准功能,避免重度定制
- 串货预警**只做规则引擎,不做 AI 预测** (一期保守,二期再升级)

### 2.2 工程项目管理模块

**核心能力**:
- 项目机会库: 录入大型工程线索 / 商机 / 项目,字段包含建筑面积 / 钢管需求量 / 业主 / 总包 / 设计院 / 当地办事处
- 销售漏斗管理: 标准七阶段 (线索 → 商机 → 投标 → 中标 → 签约 → 履约 → 回款),漏斗转化率自动统计
- 投标管理: 投标项目组成员协作 + 技术标 / 商务标文档版本管理 + 投标历史价格库 (新报价自动调出历史参考)
- 报价审批: 配置 5 级金额阶梯审批,审批节点自动派单到对应审批人,超时提醒
- 项目协作: 项目甘特图 / 任务清单 / @提醒 / 文件共享,核心干系人 (销售 / 技术 / 商务 / 生产 / 物流 / 财务) 进入项目组

**关键决策**:
- 项目甘特图采用**纷享销客标准项目看板组件**,不做高度自定义的甘特图视图 (避免性能问题)
- 投标历史价格库**只对项目组开放**,不对全员开放,降低数据泄露风险

### 2.3 终端动销采集模块

**核心能力**:
- 微信小程序: 终端门店老板扫码进入,3 步内完成日动销上报 (产品类别 / 数量 / 客户类型)
- 库存盘点: 每月一次,门店老板按提示拍照上传库存照片 + 录入数量
- 价格执行抽查: 大区销售经理出差时小程序拍照上报终端实际成交价,系统识别异常价
- 激励机制: 门店动销数据上报积分 → 兑换"友发周边"或"运费补贴"

**关键决策**:
- 首期门店覆盖**目标调整为 800 家** (从原定 1500 家下调,根据 03-22 调研结论)
- 小程序 UI 强制做**点击 3 下完成动销**的可用性测试,通过率不达 95% 不上线
- 不做"门店主动登录上报",改为**经销商主导**: 经销商业务员到门店服务时帮门店老板上报,门店积分归门店

### 2.4 客户主数据 / ERP 集成

**核心能力**:
- 客户主数据: CRM 为主数据源,ERP 为副数据源,通过中间件 EAI 实时双向同步
- 订单数据: CRM 创建 / ERP 接收,单向同步,避免双向同步循环问题
- 中间件选型: **采用客户已购的 EAI 平台 (浪潮云中间件)**,而非新采购 MuleSoft,降低成本

**关键决策**:
- 客户主数据归属问题是历史敏感问题,**项目章程必须明确"CRM 主、ERP 副"**,需 CIO 王立军在启动会拍板
- 中间件部署在客户机房,数据不出域

### 2.5 营销决策看板

- 销售漏斗看板: 全公司 / 大区 / 销售经理三个粒度的漏斗转化率
- 经销商健康度看板: 进货 / 回款 / 串货指数 / 投诉 4 维度评分
- 终端动销看板: 区域热力图 / TOP 100 门店动销曲线 / 价格异常清单

## 三、关键设计原则

1. **业务先行、技术兜底**: 一切配置先让业务方在 sandbox 跑通流程,再上生产
2. **小步快跑、迭代上线**: 6 个月内分 3 个版本上线 (M1.0 经销商 + 客户主数据,M2.0 工程项目,M3.0 终端动销),避免大爆炸式上线
3. **业务规则不写死**: 价格档级 / 审批阶梯 / 漏斗阶段都做成可配置项,客户业务自己维护
4. **数据可追溯**: 所有政策变更 / 价格变更 / 客户变更全程记录变更人 + 变更时间 + 变更前后值

## 四、组织保障建议

1. 客户方需指定**专职项目经理**,日投入 100% (现客户 PM 兼职 50% 是最大风险)
2. 设立**项目变革管理小组**,由营销总经理张志强直接领导,处理一线抗拒
3. **关键业务方深度参与**: 渠道总监李梅 / 工程总监周建华 / 零售总监刘洋,每周至少 4 小时参加项目会
4. **IT 部门储备 2 名集成人员**,配合 ERP 接口开发与运维

## 五、价值收益预估 (12 个月内)

| 指标 | 现状 | 目标 | 价值 |
|---|---|---|---|
| 经销商签约周期 | 12-18 天 | 3-5 天 | 缩短 70%,提升合作伙伴体验 |
| 大型项目跟进可视化 | 月度报表 | 实时看板 | 高管决策响应提速 3 倍 |
| 串货识别 | 客户投诉后被动发现 | T+1 主动预警 | 串货案例下降 50% (目标) |
| 终端动销可见度 | 0 | 800 家门店 | 触达终端的促销执行率提升 |
| 项目报价周期 | 2-3 周 | 5-7 天 | 投标响应速度提升,中标率预期 +15% |
