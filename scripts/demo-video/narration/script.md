# 实施工作台 · 演示视频旁白稿

> 用 mimo-v2.5-tts 模型 / 白桦 音色合成。
> 每个 `## scene-XX` 块就是一段独立旁白, 对应一段 webm。
> 用词原则: 涉及知识库/数据源/集成时统一加"当前/目前/下一步"前缀, 避免"已经全打通"的误导。

## scene-01-intro

我们用了大半年时间, 给纷享销客 CRM 实施团队做了一款专属的 AI 工作台, 我们把它叫做实施工作台。

实施顾问每天大量时间在做什么? 写启动会 PPT, 整理调研问卷, 翻历史方案找参考, 给客户出洞察报告。这些工作机械, 重复, 但又跑不掉。

这个工作台想解决的就是这件事——把顾问的工作流写进系统里, 让 AI 把机械的部分接过来。围绕项目立项、调研、蓝图、上线这条主线, 每一个阶段都提供 AI 辅助。

技术上, 我们用文档向量化做基础, 用 LLM 多智能体编排做生成, 用对抗式质量控制做兜底。 当前已实现独立工作台版本, 顾问可以自己上传文档积累知识库, 验证产品能力。

下面用大约六分钟时间, 带你走一遍核心流程。

## scene-02-documents

第一步, 把项目资料喂进来。这里是我们造的一个演示项目, 友发钢管 CRM 一期实施。 顾问把 SOW、业务调研纪要、解决方案建议书三份核心文档传到系统里。

当前阶段, 这些资料由顾问各自上传积累, 下一步我们计划接入公司方法论库和案例库, 让行业 know-how 沉淀进来。

上传后系统自动做切片和向量化处理, 整个过程在后台异步完成。

## scene-03-chunks

文档进来后会自动切片, 但切片不是简单按段落分。 我们让 LLM 按业务语义切, 每一条切片都带行业标签、文档类型、内容主题三层元数据。

顾问可以在这里人工校对每一条切片, 改错的、合并碎的、删冗余的。 这是质量的第一道关——切片质量直接决定后面所有 AI 输出的下限。

## scene-04-qa

切片处理完, 知识库就活了。

顾问可以直接对当前项目提问。 系统从切片里检索相关内容, 重新排序, 然后让 LLM 基于检索到的证据生成回答。 每一个答案都带引用脚标, 鼠标悬停可以看到原文出处, 顾问可以一键跳回原始文档核对。

这是 RAG 问答的标准能力, 我们的差异化在引用粒度——精确到文档某一段, 不是某个文件。

## scene-05-insight

接下来是核心能力——项目洞察自动生成。

打开任意一个项目, 进入项目详情。 整个工作台围绕项目阶段组织, 项目洞察、需求调研、启动会, 每个阶段都有自己的工作区和产出物。

左侧是文档清单, 必备和推荐分类清晰列出。 中央是当前阶段的工作区, 现在显示的是项目洞察的实时生成进度。 系统把项目里的 SOW、方案、合同这些核心文档整篇喂给 LLM, 不切片, 因为切片会丢上下文。

生成时间两到五分钟, 出一份按十个标准模块组织、每段都带证据溯源的项目洞察报告。

## scene-06-challenge-loop

生成不是一次性的。中央这张进度卡, 体现的是我们设计的五阶段流水线——规划、生成、评审、审视、复盘。

规划阶段先决定哪些模块要写、用什么信息源。 生成阶段逐个模块写。 评审阶段是第一层 Critic, 检查四个维度: 具体度、证据强度、时效性、下一步行动。 审视阶段是第二层 Challenger, 整篇文章对抗式审查六个维度, 发现重大问题就整篇重生成。 复盘阶段记录问题供后续学习。

这五阶段循环, 把同行评审和反方辩护内化成系统能力, 产物质量明显高于裸 LLM 输出。

## scene-07-survey

第二个核心能力, 是需求调研。

切换到需求调研阶段。 中央是需求调研工作区, 左侧是问卷的组织方式, 可以按角色看, 也可以按 LTC 流程模块看。 高管、部门负责人、一线、IT, 每类访谈对象有不同的访谈侧重。

我们走的不是发问卷给客户填字的老路, 而是顾问主导引导式访谈, 当场屏上勾选。 系统出两份产物: 一份是调研大纲, 顾问拿着访谈用, 九列日程表清晰列出每个时段聊什么; 一份是调研问卷, 结构化题目加预填选项池, 顾问聊的时候只点选, 不打字。

## scene-08-outline-challenge

工作区上方三个 tab, 切到调研大纲或调研问卷录入界面, 顾问就可以现场跟客户开会用了。

挑战流程不只是用于洞察生成。 在知识挑战页, 顾问可以对知识库里任意一条切片发起对抗式挑战——一个 agent 攻, 一个 agent 守, 最后人工判定。 通过的切片标记为高质量, 没通过的进改写队列。

这是我们对知识质量的底线兜底。

## scene-09-outro

简单总结一下。

我们做的是把咨询师从文档工里解放出来, 把更多时间还给客户。

产品路线图分三步走。当前阶段, 独立工作台已经能跑, 主要做产品力验证。 下一步, 接入公司已有的方法论库和案例库, 把行业 know-how 沉淀进来。 远期, 在大黄蜂 agent 平台上线后, 工作台作为一个 agent 接入, 探索在一线销售和顾问场景里的落地路径。
