# APL质量门控定义

## 门控级别

### 阻断级(Blocker)
必须通过,否则不允许提交代码。

### 警告级(Warning)
建议修复,但不阻断提交。

## 门控检查项

### 1. 错误处理完整性 [阻断级]

**检查内容**:
- 所有API调用都有错误处理
- 错误日志包含详细信息
- 用户友好的错误提示

**检查方法**:
```groovy
// 检查是否有未处理错误的API调用
// 搜索模式: Fx\.(object|http|message|function)\.\w+\([^)]*\)(?!.*if\s*\(.*error)
```

**通过标准**:
- 所有Fx.* API调用都有 `if (error)` 判断
- 错误处理包含日志记录
- 错误提示对用户友好

**不通过后果**: 阻断提交,必须修复

### 2. 日志记录完整性 [阻断级]

**检查内容**:
- 关键步骤有日志记录
- 错误日志包含上下文信息
- 使用合适的日志级别

**检查方法**:
```groovy
// 检查关键操作是否有日志
// 1. 函数入口是否有日志
// 2. 错误处理是否有日志
// 3. 关键业务逻辑是否有日志
```

**通过标准**:
- 函数入口有info日志
- 错误处理有error日志
- 关键步骤有info日志

**不通过后果**: 阻断提交,必须补充日志

### 3. 平台限制检查 [阻断级]

**检查内容**:
- API调用次数是否超限
- 执行时间是否可能超限
- 内存使用是否合理

**检查方法**:
```groovy
// 1. 统计API调用次数
// 2. 评估执行时间
// 3. 检查是否有批量操作
```

**通过标准**:
- Fx.object调用 ≤ 300次
- Fx.http调用 ≤ 50次
- 使用批量操作代替循环单条操作

**不通过后果**: 阻断提交,必须优化

### 4. 命名规范 [警告级]

**检查内容**:
- API名称中不能包含下划线（除了结尾的 `__c`）
- 变量名使用驼峰命名法
- 常量名使用大写字母和下划线
- 避免使用保留字

**检查方法**:
```groovy
// 检查API命名
// 1. 检查API名称是否包含下划线（除了结尾的__c）
// 2. 检查变量命名是否使用驼峰命名法
// 3. 检查是否使用了保留字(owner, this, delegate)
```

**通过标准**:
- API名称不含下划线（除了结尾的 `__c`）
  - ✅ 正确：`EvtMatchContact__c`
  - ❌ 错误：`Evt_MatchContact__c`
- 变量名符合驼峰命名法
- 未使用保留字

**不通过后果**: 警告提示,建议修复

### 5. 性能优化 [警告级]

**检查内容**:
- 使用批量操作
- 避免重复查询
- 控制查询字段数量

**检查方法**:
```groovy
// 1. 检查是否有循环单条操作
// 2. 检查是否有重复查询
// 3. 检查查询字段数量
```

**通过标准**:
- 使用batchCreate/batchUpdate代替循环create/update
- 查询字段数量合理

**不通过后果**: 警告提示,建议优化

### 6. 安全检查 [阻断级]

**检查内容**:
- 验证输入参数
- 检查数据权限
- 不记录敏感信息

**检查方法**:
```groovy
// 1. 检查参数验证
// 2. 检查权限检查
// 3. 检查日志内容
```

**通过标准**:
- 必填参数有验证
- 敏感操作有权限检查
- 日志不包含敏感信息

**不通过后果**: 阻断提交,必须修复

## 门控执行流程

### Step 1: 代码扫描
自动扫描代码,检查以下内容:
- API调用次数
- 错误处理覆盖率
- 日志记录覆盖率
- 命名规范符合度

### Step 2: 问题分类
将发现的问题分类:
- 阻断级问题: 必须修复
- 警告级问题: 建议修复

### Step 3: 结果输出
输出检查结果:
```
质量门控检查结果:
==================

✅ 错误处理完整性: 通过
✅ 日志记录完整性: 通过
❌ 平台限制检查: 未通过
   - Fx.object调用次数: 350次 (超限50次)
   - 建议: 使用批量操作优化

⚠️ 命名规范: 警告
   - 变量 'n' 命名不清晰
   - 建议: 使用更具描述性的名称

✅ 性能优化: 通过
✅ 安全检查: 通过

总结: 1个阻断级问题, 1个警告级问题
```

### Step 4: 门控决策
- 所有阻断级问题通过 → 允许提交
- 存在阻断级问题 → 阻断提交,提示修复

## 门控配置

### 可配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| maxObjectCalls | 300 | Fx.object最大调用次数 |
| maxHttpCalls | 50 | Fx.http最大调用次数 |
| maxFunctionCalls | 50 | Fx.function最大调用次数 |
| maxMessageCalls | 50 | Fx.message最大调用次数 |
| requireErrorHandling | true | 是否要求错误处理 |
| requireLogging | true | 是否要求日志记录 |
| checkNamingConvention | true | 是否检查命名规范 |

### 场景特定配置

不同场景可以有不同的门控标准:

| 场景 | 执行时间限制 | 特殊要求 |
|------|-------------|---------|
| 按钮开发 | 20秒 | 必须返回结果 |
| 流程开发 | 300秒 | 必须返回Boolean |
| 计划任务 | 600秒 | 必须有批量处理 |
| 自定义控制器 | 300秒 | 必须有参数验证 |
| 事件监听 | 300秒 | 必须有异常处理 |
