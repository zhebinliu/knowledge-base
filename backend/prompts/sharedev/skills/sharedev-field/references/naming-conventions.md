# 字段命名规范

## 预置字段
- 格式：`camelCase`
- 示例：`name`、`owner`、`createTime`、`lastModifyTime`
- 标准字段由平台定义，**不可新建同名字段**

## 自定义字段
- 格式：`field_<id>__c`
- `<id>` 为 5 位字母数字混合标识符
- 示例：`field_2l23J__c`、`field_aB9kf__c`
- ID 生成：随机 5 位 [a-zA-Z0-9] 字符串，生成后需确认不与已有字段冲突

## 通用规则

1. API Name 一旦创建后**不可更改**
2. `__c` 后缀仅用于自定义配置，标准配置无此后缀
3. ID 部分区分大小写
4. 同一作用域内 API Name 必须唯一（字段在对象内唯一）
