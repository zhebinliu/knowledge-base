# APL 代码审查检查清单

## 1. 命名规范检查

- [ ] API名称不能包含下划线（除了结尾的 `__c`）
  - ✅ 正确：`EvtMatchContact__c`, `BtnUpdateAccount__c`
  - ❌ 错误：`Evt_MatchContact__c`, `Btn_UpdateAccount__c`
- [ ] 变量名使用驼峰命名法（camelCase）
- [ ] 常量名使用大写字母和下划线（UPPER_SNAKE_CASE）
- [ ] 函数名清晰表达功能意图
- [ ] 避免使用保留字（owner, this, delegate）

## 2. 错误处理检查

- [ ] 所有 Fx.object 调用都有错误处理
- [ ] 所有 Fx.http 调用都有错误处理
- [ ] 错误日志包含详细信息
- [ ] 用户友好的错误提示

## 3. 日志记录检查

- [ ] 函数入口有日志记录
- [ ] 关键步骤有日志记录
- [ ] 错误情况有日志记录
- [ ] 不记录敏感信息到日志

## 4. 性能优化检查

- [ ] 使用批量操作代替循环单条操作
- [ ] 避免重复查询相同数据
- [ ] 控制查询返回字段数量
- [ ] 合理使用 limit 限制结果集

## 5. 安全检查

- [ ] 验证输入参数
- [ ] 检查数据权限
- [ ] 不记录敏感信息到日志
- [ ] 不在代码中硬编码密钥

## 6. 平台限制检查

- [ ] Fx.object 调用不超过 300 次/函数
- [ ] Fx.http 调用不超过 50 次/函数
- [ ] 执行时间不超过限制（按钮20秒/流程300秒/计划任务600秒）
